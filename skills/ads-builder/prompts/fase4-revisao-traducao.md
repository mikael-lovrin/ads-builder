# Fase 4 — Revisão e Tradução (opcional)

Esta fase só roda sob pedido explícito do usuário — "avalie o que escrevemos" ou "traduza para [mercado]". Não é parte obrigatória do fluxo principal.

---

## Modo: Avaliação

Quando o usuário pedir para avaliar um roteiro/briefing finalizado (próprio ou de terceiros):

Avalie com rigor, sem suavizar:
- Uma nota para cada gancho gerado
- Uma nota para cada bloco da estrutura (ver `knowledge/structure-library.md` ou `knowledge/static-image-psychology.md`, conforme o tipo)
- Uma nota final consolidada
- Aponte inconsistências explicitamente: promessa do gancho que o corpo não cumpre, vilão genérico demais, storytelling micro que não segue a regra das 2 frases, número/claim que não veio do briefing real, CTA fraco ou fora da sequência White→Gray→Black

Formato:
```markdown
## Avaliação — [nome do criativo]

### Ganchos
- Gancho A: [nota]/10 — [por quê]
...

### Estrutura
- [Bloco]: [nota]/10 — [por quê]
...

### Nota Final: [nota]/10

### Inconsistências Encontradas
- [item específico, onde está, o que está errado, como corrigir]
```

## Modo: Tradução / Localização de Mercado

Quando o mercado-alvo definido na Fase 0 for diferente do idioma desta conversa, ou quando o usuário pedir explicitamente para traduzir um criativo já aprovado:

Regras:
- Não é tradução literal — é localização de voz. Mantenha a estrutura e a lógica argumentativa intactas, mas escreva como se fosse originalmente criado por uma pessoa nativa do mercado-alvo
- Pense no texto como roteiro de teleprompter — pausas naturais, ritmo de fala, não de texto escrito
- Persona de referência para mercados anglófonos de saúde/beleza: "relatable influencer" — alguém que parece uma amiga dando um conselho honesto, não uma apresentadora corporativa
- Use termos comuns do nicho naquele mercado específico (ex.: EUA saúde/beleza: "leaning out", "through the roof", "game changer") — pesquise se não tiver certeza do termo nativo
- Não aplique formatação especial (negrito, itálico) nem cite o texto entre aspas — é para ser lido/decorado, não publicado como está

Salve a versão localizada substituindo (ou adicionando, se o usuário quiser manter as duas) no documento de origem (`roteiros-video.md` ou `briefing-imagem.md`), com uma nota no topo indicando o idioma/mercado de cada versão.
